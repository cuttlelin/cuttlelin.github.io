一、 前言
現今餐廳在路上琳瑯滿目，讓消費者眼花撩亂，下一餐要吃甚麼永遠都是個未知數，況且僅僅滿足口腹之慾早已不是人們的目的，再加上科技的進步，人手一機，我們想開發一款人人可用的APP，滿足消費者的需求以及餐廳業者的目的。

二、 創意描述
我們想透過結合雙方的需求，達到雙贏的局面，店家希望吸引更多顧客上門，而消費者希望有優惠或是動機去嘗試不同的餐廳，我們希望加入 ”任務”和”優惠”的元素，透過我們的APP，餐廳會提供任務和優惠，而消費者接取任務，到指定的餐廳消費，並完成得到優惠獎勵，在這中間消費者會為了完成任務而去消費特定的餐廳，餐廳得到了客流量，而消費者得到優惠。

三、 系統功能簡介
有與我們合作的店家，會提供任務以及獎勵，而消費者按下接取任務，會隨機接取500公尺內有與我們合作店家的任務，完成任務內容後，請店家輸入專屬該店家的驗證碼，即算完成了任務，完成一定數量的任務，便會獲得隨機獎勵。在獎勵的使用期限內，在此獎勵所屬的餐廳，出示該獎勵視窗，並按下確認，以完成兌換。除此之外，也可以透過我們的地圖去查看或搜尋想探索之地方的餐廳，或是篩選自己想找的餐廳種類，點選餐廳標記後，便可看到該餐廳的各項詳細資訊，亦可以瀏覽別人對此餐廳的評論，或是寫下評論。

四、 系統特色
不論有沒有和我們合作，消費者都可以透過我們的APP查詢到，對使用者來說非常方便，然而與我們合作的餐廳，則可以大大的提升自己的曝光率。加入了遊戲性的元素，讓用餐這項例行公事多了趣味性，或是遲遲無法決定要吃甚麼?也可以交給命運，讓我們的APP幫你決定，去探索發現一些從未品嘗過的餐廳。

五、 系統開發工具與技術
1. Android Studio
Android Studio是開發在Android平台上的APP非常方便且容易上手的開發平台，而且是免費使用。人性化的介面加上非常多人使用，所以在查詢書籍或是網路資料都很方便。
2. Google API
Google的API庫提供非常多各式功能的API，其中我們用了GOOGLE MAP API、GOOGLE PLACE API以及GOOGLE WEB SEVER，有了這些API讓我們在使用各項技術事半功倍，且GOOGLE API的文件寫得非常易懂，讓從零開始學習的我們非常順利。
3. Facebook API
Facebook提供了許多的API讓開發人員使用，其中我們用了facebook的登入API，讓已有facebook帳號的人在使用我們的app時，能夠省去建立帳號的時間，直接以facebook帳號登入app。
4. AppServ
WebServer開發工具，內建Appache、MySQL，使用PHP做資料庫的連接，方便我們使用。

六、 系統使用對象
使用者可以是一般的APP使用者，也就是所有人，還有餐廳業者，餐廳業者可以透過註冊餐廳來和我們合作。

七、 系統使用環境
作業系統與版本 Android 5.1版本以上
儲存空間 100MB以上
記憶體空間 1G以上
八、 結語
第一次開發APP，對比以前的網站或是系統開發，有滿多的差異，像是要更加關心使用者的觀感，因為螢幕縮小到手機的畫面，很多功能介面必須簡化，讓使用者一目了然。由於是第一次使用Android studio開發APP，一開始就花了大概2.3個禮拜去看書，大概了解架構後實際要運用起來卻又覺得太基本了，為了有更完整及更好看的介面，又花了很多時間上網爬文章，而網路上的文章多如牛毛，真正符合自己需求的又是少之又少，這是一個滿大的瓶頸。
最困難的部份應該是地圖吧，這是我們系統的核心功能之一，地圖功能最多人使用的當然是GOOGLE API，但這也是第一次使用公開的API，所以在查詢資料及觀看官方文件上花了很多功夫。關於我們要的餐廳資訊，在GOOGLE MAP ANDROID裡面其實並沒有，因此遇到了一個瓶頸，就是我們得把餐廳資料一筆一筆的KEY進資料庫中，但是這太困難了，在一番折騰的爬文及嘗試後，發現GOOGLE MAP WEB的服務中，有可以查詢各地點的詳細資訊這項服務，所以我們決定用此服務來篩選成餐廳，運用成我們所想要的資料。
